# Codebase Structure

**Analysis Date:** 2024-03-28

## Directory Layout

```
D:\projects\spoon\
├── .claude/                # Claude Code configuration
├── .claude-plugin/         # Plugin manifest and hooks
├── .planning/             # Planning documents
├── contrib/               # External contributions
├── scripts/               # Shared PowerShell scripts
├── skills/                # Domain-specific skills
├── spoon/                 # Rust CLI application
├── spoon-backend/        # Backend service abstractions
├── spoon.exe              # Compiled binary
├── xtask/                 # Build and deployment utilities
└── target/                # Build artifacts
```

## Directory Purposes

**`spoon/` (Rust CLI Application):**
- Purpose: Main executable with CLI and TUI
- Contains: Source code for spoon.exe
- Key files:
  - `src/main.rs` - Entry point
  - `src/lib.rs` - Library exports
  - `src/cli/` - Command parsing and execution
  - `src/service/` - Application logic
  - `src/tui/` - Terminal user interface
  - `src/config/` - Configuration management

**`spoon-backend/` (Service Backend):**
- Purpose: Core service abstractions and event system
- Contains: Backend traits and implementations
- Key files:
  - `src/lib.rs` - Backend exports
  - `src/event.rs` - Event definitions
  - `src/scoop.rs` - Scoop backend service
  - `src/msvc.rs` - MSVC backend service
  - `src/task.rs` - Async task utilities

**`skills/` (Domain Skills):**
- Purpose: Extensible tool management domains
- Contains: Scoop, proxy, ai-toolchain, python-via-uv
- Structure per skill:
  - `[skill]/` - Skill directory
  - `[skill]/SKILL.md` - Skill documentation
  - `[skill]/SKILL-zh.md` - Chinese documentation
  - `[skill]/references/` - Skill-specific references

**`contrib/` (External Contributions):**
- Purpose: External tool integrations and utilities
- Contains: hok (libscoop), msvcup, Scoop source
- Note: Independent projects integrated with spoon

**`scripts/` (Shared Scripts):**
- Purpose: Reusable PowerShell utilities
- Contains: Path management and command execution
- Key files:
  - `add-path.ps1` - Add to PATH
  - `run-cmd.ps1` - Execute PowerShell commands

**`xtask/` (Build System):**
- Purpose: Build and deployment automation
- Contains: Release deployment utilities
- Key features: Hot deployment, artifact management

## Key File Locations

**Entry Points:**
- `spoon.exe` - Main executable
- `spoon/src/main.rs` - Application entry point
- `spoon-backend/src/lib.rs` - Backend entry point
- `xtask/src/main.rs` - Build system entry point

**Configuration:**
- `.claude/settings.json` - Plugin hooks
- `spoon/src/config/` - Application config management

**Core Logic:**
- `spoon/src/cli/` - Command parsing and routing
- `spoon/src/service/` - Service layer implementation
- `spoon/src/actions/` - Tool actions and execution

**Testing:**
- `spoon/tests/` - Unit tests
- `spoon-backend/tests/` - Backend tests

## Naming Conventions

**Files:**
- `*.rs` - Rust source files
- `SKILL*.md` - Skill documentation
- `*.toml` - Configuration files
- `*.json` - JSON configuration
- `*.ps1` - PowerShell scripts

**Directories:**
- `snake_case` - Rust modules
- `kebab-case` - Documentation files
- `PascalCase` - Module directories (spoon, spoon-backend)
- `lowercase` - Skill directories

**Variables/Functions:**
- `snake_case` for Rust code
- `camelCase` for CLI arguments
- `TitleCase` for types and structs

## Where to Add New Code

**New Skill:**
- Primary code: `skills/[skill-name]/`
- Tests: `spoon/tests/` (if service-layer integration)
- Documentation: `skills/[skill-name]/SKILL.md`

**New Backend Service:**
- Implementation: `spoon-backend/src/[service].rs`
- Tests: `spoon-backend/tests/`
- Integration: `spoon/src/service/mod.rs`

**New Command:**
- CLI definition: `spoon/src/cli/`
- Service logic: `spoon/src/service/`
- Actions: `spoon/src/actions/`

**New Configuration:**
- Model: `spoon/src/config/model.rs`
- I/O: `spoon/src/config/io.rs`
- Usage: `spoon/src/service/mod.rs`

## Special Directories

**`contrib/`:**
- Purpose: External tool contributions
- Generated: No
- Committed: Yes
- Integration: Read-only, updated via external sources

**`.planning/`:**
- Purpose: Generated documentation
- Generated: Yes
- Committed: Yes
- Maintenance: Auto-generated by GSD commands

**`target/`:**
- Purpose: Build artifacts
- Generated: Yes
- Committed: No
- Cleanup: Added to .gitignore

**`skills/` subdirectories:**
- Purpose: Tool-specific skills
- Generated: No
- Committed: Yes
- Pattern: Each skill self-contained with docs
