<objective>
Migrate 5 app modules from app-side config path helpers to backend-owned RuntimeLayout, eliminating parallel path derivation for BNDR-04 and LAY-01.

Purpose: The verification found that 5 app modules still call config::scoop_root_from, config::msvc_root_from, config::shims_root_from, config::msvc_cache_root_from, and config::msvc_toolchain_root_from instead of RuntimeLayout::from_root. These parallel derivation paths contradict the single-backend-ownership goal: changing root does not consistently update all behavior through backend layout alone. This plan migrates all production call sites to use RuntimeLayout.

Output: All 5 app files using RuntimeLayout for backend path derivation; no remaining production uses of config backend path helpers in these files.
</objective>
