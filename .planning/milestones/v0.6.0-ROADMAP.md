# Milestone v0.6.0: Backend Architecture Completion

**Status:** SHIPPED 2026-04-01
**Phases:** 06-09

## Overview

This milestone completed the second major backend cleanup arc: it turned MSVC into a first-class backend domain with explicit seams, canonical SQLite-backed state, shared lifecycle ownership, hardened shared backend contracts, and a focused regression net to protect the result.

## Milestone Summary

### Key Accomplishments

1. Turned MSVC into a first-class backend domain with explicit seams and thin app-shell ownership.
2. Added canonical MSVC state to the SQLite control plane and made lifecycle execution update it for both managed and official strategies.
3. Reset and hardened the backend contracts shared across Scoop and MSVC: events, errors, host/path contracts, and shared IO primitives.
4. Added backend-heavy regression coverage for the riskiest canonical-state, lifecycle, and shared-contract failure boundaries.

### Key Decisions

- Treat `managed` and `official` as runtime strategies inside one MSVC domain rather than parallel products.
- Keep canonical state authoritative, with detection/validation as evidence and reconcile source.
- Use forward design for shared backend contract hardening instead of compatibility preservation.
- Keep `fsx` narrow and extract only shared primitives, not one giant runtime abstraction.

### Issues Resolved

- MSVC no longer lives as a backend domain with partial ownership and scattered state/lifecycle semantics.
- Shared event/error/path contracts are no longer as weak or ad hoc as they were after the first milestone.
- The most dangerous canonical-state/lifecycle failure boundaries are now regression-tested.

### Issues Deferred

- Further `fsx` consolidation
- Additional backend error-contract tightening
- Lower-value derive-not-store cleanup in read models
- Full `SystemPort` / `ScoopRuntimeHost` simplification
- Event follow-up seed and `async_zip` evaluation seed
- Backlog test-architecture cleanup

### Technical Debt

- `spoon` still carries a small set of non-blocking warnings around deprecated path helpers and a few unused imports/variables.
- Shared archive handling is improved but still not fully unified: 7z still depends on an external helper path.
- Ignored real managed MSVC validate smoke remains environment-dependent.
