# Milestone v0.5.0: Spoon Backend Refactoring

**Status:** SHIPPED PREPARED 2026-03-31

## Overview

This milestone turned `spoon` into a much thinner app shell over `spoon-backend` by first tightening backend ownership, then consolidating Scoop state, then moving the control plane to SQLite, then splitting Scoop lifecycle behavior into explicit backend phases, and finally hardening the refactor with targeted regression coverage and gap-closure work.

## Milestone Summary

### Key Accomplishments

1. Established backend-owned context, layout, Git, and app/backend seams so `spoon` no longer owns backend runtime internals.
2. Replaced duplicate Scoop installed-state models with one canonical backend-owned state model.
3. Moved the control plane to SQLite while preserving the filesystem as the runtime data plane.
4. Split Scoop install/update/uninstall/reapply into explicit backend lifecycle modules with shared stage semantics.
5. Added targeted safety-net coverage across backend failure boundaries, control-plane diagnostics, CLI/TUI translation, and milestone gap-closure regressions.

### Key Decisions

- Keep `spoon` as a thin app shell and `spoon-backend` as the single trusted backend core.
- Use forward design instead of preserving low-quality legacy compatibility layers.
- Treat SQLite as control plane and the filesystem as data plane.
- Keep ordinary logs in `tracing`; structured backend events carry product semantics.
- Preserve `reapply` as a distinct lifecycle concept rather than collapsing it into reinstall.

### Issues Resolved

- Duplicate Scoop state models and stale flat JSON control-plane assumptions.
- Mixed ownership between app code and backend runtime/lifecycle behavior.
- Large lifecycle controller behavior concentrated in a monolithic Scoop runtime path.
- Safety-net gaps where tests no longer matched the shipped SQLite/canonical contract.

### Issues Deferred

- Test architecture cleanup as a backlog item (`999.1`).
- Backend event contract hardening as a seed.
- Backend error contract tightening and production hardcoded-path cleanup as todos.

### Technical Debt

- `spoon` still carries a small set of non-blocking warnings around deprecated path helpers and a few unused imports/variables.
- Ignored real remote Scoop smoke tests remain environment-dependent and best-effort.
